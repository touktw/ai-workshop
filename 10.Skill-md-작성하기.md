# 10단계: Skill.md 작성하기

> 이 챕터에서 할 것: 자주 쓰는 AI 명령을 Skill 파일로 저장해서 빠르게 불러 쓰는 법을 배웁니다.

> ⚠️ 먼저 [6단계: Claude Code 시작하기](06.Claude-Code-시작하기.md)를 완료해야 합니다.

---

## 10-1. Skill 파일이란?

Claude Code를 쓰다 보면 같은 요청을 반복하게 됩니다.  
예를 들어 "이 코드를 쉽게 설명해줘", "버그를 찾아줘" 같은 요청을 매번 타이핑하는 것은 번거롭습니다.

**Skill 파일은 자주 쓰는 요청을 파일로 저장해두고, 짧은 명령어로 불러 쓰는 기능**입니다.  
`/코드-설명` 처럼 슬래시(/) 뒤에 파일 이름을 입력하면 저장해둔 내용이 자동으로 실행됩니다.

---

## 10-2. Skill 파일 저장 위치

Skill 파일은 프로젝트 폴더 안의 `.claude/commands/` 폴더에 저장합니다.

1. Cursor에서 내 프로젝트 폴더(`my-todo-app`)를 엽니다.
2. 왼쪽 파일 목록에서 빈 곳을 오른쪽 클릭 → **New Folder**를 선택합니다.
3. 폴더 이름을 `.claude`로 입력합니다.

> 💡 `.`으로 시작하는 폴더는 숨김 폴더입니다. 파일 목록에서 안 보일 수 있습니다.

4. `.claude` 폴더 안에 같은 방법으로 `commands` 폴더를 만듭니다.

---

## 10-3. 첫 번째 Skill 만들기

`.claude/commands/` 폴더 안에 `코드-설명.md` 파일을 만들고 아래 내용을 입력하세요:

```markdown
이 코드가 어떻게 작동하는지 초등학생도 이해할 수 있게 한국어로 설명해 주세요.
전문 용어 없이, 일상적인 비유를 사용해서 설명해 주세요.
```

`Command(⌘) + S`로 저장합니다.

---

## 10-4. Skill 사용하기

터미널에서 프로젝트 폴더로 이동해서 Claude Code를 실행합니다:

```bash
cd ~/Desktop/my-todo-app
claude
```

Claude Code 대화창에 아래처럼 입력하면 저장해둔 Skill이 실행됩니다:

```
/코드-설명
```

> 💡 슬래시(`/`) 뒤에 파일 이름(확장자 제외)을 입력하면 됩니다.

---

## 10-5. 유용한 Skill 예시

아래 내용을 참고해서 나만의 Skill을 만들어 보세요.

**버그 찾기** (`.claude/commands/버그-찾기.md`)
```markdown
이 코드에서 버그나 오류가 발생할 수 있는 부분을 찾아주세요.
발견된 문제는 번호를 매겨 하나씩 설명해 주세요.
```

**기능 추가** (`.claude/commands/기능-추가.md`)
```markdown
아래에 설명하는 기능을 추가해 주세요.
기존 코드 스타일을 유지하고, 수정한 부분을 간략히 설명해 주세요.
```

**코드 개선** (`.claude/commands/코드-개선.md`)
```markdown
이 코드를 더 읽기 쉽고 효율적으로 개선해 주세요.
개선한 이유를 각 변경 사항마다 설명해 주세요.
```

## FAQ

**Q. Skill 파일 이름에 한글을 써도 되나요?**  
네, 한글 파일 이름도 됩니다. `/코드-설명` 처럼 그대로 입력하면 됩니다.

**Q. `.claude` 폴더가 파일 목록에 안 보여요.**  
숨김 폴더라서 기본적으로 안 보일 수 있습니다. Cursor에서 `Command(⌘) + Shift + .`을 누르면 숨김 파일을 볼 수 있습니다.

**Q. Skill을 수정하면 바로 적용되나요?**  
네, 저장 후 다음 `/명령어` 호출 시 바로 적용됩니다.
